# MCP 通用客户端配置指南

本文档说明如何将 Mindmap VSCode 的 MCP 服务器配置到任意 MCP 兼容客户端（Claude Desktop、Codex CLI、Cline 等）。

## 推荐策略

- **新用户、MCP 自动化与日常独立编辑**：以 **Mindmap Desktop（Electron）** 为**首选**；启动桌面应用即可使用内置 HTTP 桥，**不必**安装 VS Code / Cursor 扩展。
- **VS Code / Cursor 扩展**：**仍保留源码与构建**（`build.py` / `install.py` 可继续产出 VSIX），供既有用户与 IDE 内嵌场景；文中扩展相关步骤为**兼容路径**。桌面与扩展**勿同时占用**默认端口 `58741`（二选一常驻）。

## 概述

### MCP 架构

Mindmap VSCode 使用标准的 MCP (Model Context Protocol) stdio 协议，通过两层桥接实现 AI 客户端与脑图编辑器的通信：

```
AI客户端（Claude/Codex/Cline）
    ↓ MCP stdio 协议
MCP进程（mcp-server/dist/index.js）
    ↓ HTTP POST /mcp-bridge/v1/call
HTTP桥接（Mindmap Desktop 主进程 或 VS Code/Cursor 扩展，127.0.0.1:58741）
    ↓ postMessage
Webview脑图（jsMind画布）
```

### 支持的客户端

- **Claude Desktop**：Anthropic 官方桌面应用
- **Codex CLI**：命令行 AI 助手
- **Cline**：VS Code 扩展
- **其他**：任何支持 MCP stdio 协议的客户端

### 前置条件

1. **HTTP 桥来源（二选一）**  
   - **推荐**：**Mindmap Desktop** 已启动并保持主窗口打开（桥见下文「Mindmap Desktop」）。  
   - **兼容**：在 VS Code / Cursor 中安装并启用本扩展，且 **`mindmap.mcpBridge.enable`** 为启用。
2. **Node.js 在 PATH 中**：MCP 服务器（`mcp-server`）需本机可执行 `node`。
3. **`mcp-server/dist/index.js` 可访问**：开发见仓库 `mindmap_vscode/mcp-server/dist/index.js`；安装包见下文「Mindmap Desktop」；若走扩展路径则见下文「查找 MCP 服务器路径」。

### Mindmap Desktop（独立桌面应用）

自桌面版起，**HTTP MCP 桥内置在 Electron 主进程**（`desktop/mcpBridge.js` 仅负责监听；与扩展共用编译产物 `dist/shared/mcpBridgeCore.js`，源 `src/shared/mcpBridgeCore.ts`；`POST /mcp-bridge/v1/call`），**无需打开 IDE**；与上文「推荐策略」一致，为 MCP 与独立编辑的**首选宿主**。

1. **启动 Mindmap Desktop**，保持主窗口打开（桥随应用启动，默认 `127.0.0.1:58741`，与扩展共用端口时请只开其一，或通过环境变量改端口）。
2. **获取 URL / Token**  
   - 在窗口**顶部区域右键**（与「显示/隐藏原生菜单栏」同一菜单）→ **「复制 MCP 连接信息（到剪贴板）」**。  
   - Token 持久化在应用 userData 下的 `mindmap-desktop-mcp-token.txt`（首次启动自动生成）。
3. **MCP 进程路径**  
   - **开发**：仓库内 `mindmap_vscode/mcp-server/dist/index.js`。  
   - **安装包**：`resources/mindmap-app/mcp-server/dist/index.js`（相对安装目录；Windows 下通常在 `MindmapDesktop x.x.x.exe` 同级 `resources` 内）。
4. **端口环境变量**（可选）：`MINDMAP_BRIDGE_PORT` 或 `MINDMAP_DESKTOP_BRIDGE_PORT`（优先使用后者覆盖）。

`get_editor_state` 返回的 `editor` 字段为 **`mindmap-desktop`**，便于与扩展场景区分。

## 快速配置步骤

### 1. 获取桥接 URL 和 Token

**方法 1：Mindmap Desktop（推荐）**

1. 启动 **Mindmap Desktop**，保持主窗口打开。  
2. 在窗口 **顶部区域右键**（与「显示/隐藏原生菜单栏」同一菜单）→ **「复制 MCP 连接信息（到剪贴板）」**。  
3. 剪贴板中为两行环境变量，例如：  
   ```
   MINDMAP_BRIDGE_URL=http://127.0.0.1:58741
   MINDMAP_BRIDGE_TOKEN=<auto-generated-token>
   ```

**方法 2：VS Code / Cursor 命令（兼容；需已安装扩展）**

1. 在 VS Code/Cursor 中打开命令面板（`Ctrl+Shift+P` / `Cmd+Shift+P`）  
2. 搜索并执行：**`Mindmap: Show MCP Bridge Info`**  
3. 环境变量会自动复制到剪贴板（格式同上）。

**方法 3：扩展设置（兼容）**

打开 VS Code 设置（`Ctrl+,` / `Cmd+,`），搜索 `mindmap.mcpBridge`：  
- `mindmap.mcpBridge.port`：桥接端口（默认 58741）  
- `mindmap.mcpBridge.token`：认证 token（空则自动生成）  
- `mindmap.mcpBridge.enable`：是否启用桥接（默认 true）

### 2. 查找 MCP 服务器路径

以下适用于**已安装扩展**、希望 `args` 指向扩展目录下 `mcp-server` 的场景。**仅用桌面版时**，优先使用**仓库** `mindmap_vscode/mcp-server/dist/index.js` 或**安装包**内 `resources/mindmap-app/mcp-server/dist/index.js`（见上文「Mindmap Desktop」）。

若由扩展安装目录引用，路径格式为：

**Windows**：
```
%USERPROFILE%\.vscode\extensions\av-ai-dev.mindmap-vscode-<VERSION>\mcp-server\dist\index.js
```

**macOS**：
```
~/.vscode/extensions/av-ai-dev.mindmap-vscode-<VERSION>/mcp-server/dist/index.js
```

**Linux**：
```
~/.vscode/extensions/av-ai-dev.mindmap-vscode-<VERSION>/mcp-server/dist/index.js
```

**提示**：
- `<VERSION>` 是扩展版本号（如 `0.0.158`），可在扩展管理器中查看
- 如果使用 Cursor，路径中的 `.vscode` 替换为 `.cursor`
- JSON 配置中统一使用正斜杠 `/`（跨平台兼容）

**快速查找**：
```bash
# Windows (PowerShell)
ls "$env:USERPROFILE\.vscode\extensions\av-ai-dev.mindmap-vscode-*\mcp-server\dist\index.js"

# macOS/Linux
ls ~/.vscode/extensions/av-ai-dev.mindmap-vscode-*/mcp-server/dist/index.js
```

### 3. 配置 MCP 客户端

根据你使用的客户端，参考下方对应的配置示例。

## 客户端配置示例

### Claude Desktop

**配置文件位置**：

- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`
- **Linux**: `~/.config/Claude/claude_desktop_config.json`

**配置内容**：

```json
{
  "mcpServers": {
    "mindmap-vscode": {
      "command": "node",
      "args": [
        "/Users/yourname/.vscode/extensions/av-ai-dev.mindmap-vscode-0.0.158/mcp-server/dist/index.js"
      ],
      "env": {
        "MINDMAP_BRIDGE_URL": "http://127.0.0.1:58741",
        "MINDMAP_BRIDGE_TOKEN": "your-token-here"
      }
    }
  }
}
```

**注意**：
- 将 `/Users/yourname/...` 替换为实际路径
- 将 `0.0.158` 替换为实际版本号
- 将 `your-token-here` 替换为步骤 1 获取的 token
- 使用正斜杠 `/`（即使在 Windows 上）

## 两层桥接机制详解

### 为什么需要两层桥接？

1. **MCP 标准要求 stdio 协议**：AI 客户端通过标准输入输出与 MCP 服务器通信
2. **VS Code 扩展与 Webview 通信需要 postMessage**：浏览器安全模型限制
3. **HTTP 桥接提供统一 API**：支持多个 MCP 客户端同时连接
4. **本机限制确保安全**：仅监听 `127.0.0.1`，不暴露到网络

### 第一层桥接：MCP 进程 ↔ HTTP 桥接

**MCP 进程**（`mcp-server/dist/index.js`）：
- 通过 stdio 与 AI 客户端通信
- 接收工具调用请求（`get_editor_state`、`batch_get`、`batch_design`）
- 通过 HTTP POST 转发到扩展内桥接
- 端点：`http://127.0.0.1:58741/mcp-bridge/v1/call`
- 认证：使用 token 验证

**HTTP 桥接**（`src/bridge.ts`）：
- 在扩展激活时启动 HTTP 服务器
- 监听 `127.0.0.1:58741`（端口可配置）
- 验证请求中的 token
- 分发到当前聚焦的 MindmapPanel

### 第二层桥接：HTTP 桥接 ↔ Webview

**MindmapPanel**（`src/panel.ts`）：
- 管理 Webview 生命周期
- 通过 `postMessage` 与 Webview 通信
- 调用 Webview 中的 AI 命令（`aiGetTree`、`aiApplyOps`、`aiGetSelection`）

**Webview**（`media/webview-app.js`）：
- 基于 jsMind 的脑图画布
- 执行实际的树操作（增删改查）
- 序列化结果返回给面板

### 完整调用流程示例

**读取脑图**：
```
1. AI 调用 batch_get(readDepth=3)
   ↓
2. MCP 进程接收 stdio 请求
   ↓
3. MCP 进程 POST http://127.0.0.1:58741/mcp-bridge/v1/call
   Body: {token, method: "batch_get", arguments: {readDepth: 3}}
   ↓
4. HTTP 桥接验证 token，调用 panel.aiGetTree()
   ↓
5. Panel postMessage 到 Webview
   ↓
6. Webview 返回树结构
   ↓
7. 按 readDepth 截断，返回给 MCP 进程
   ↓
8. MCP 进程通过 stdio 返回给 AI 客户端
```

## 可用的 MCP 工具

### get_editor_state

**功能**：获取编辑器状态

**参数**：
- `include_schema`（必填，布尔）：是否返回操作说明

**返回**（字段随宿主变化）：
- **`editor`**：**Mindmap Desktop** 下为 **`mindmap-desktop`**；**VS Code / Cursor 扩展** 下为 **`mindmap-vscode`**。

```json
{
  "active": true,
  "editor": "mindmap-desktop",
  "title": "测试脑图",
  "filePath": "/path/to/mindmap.jm",
  "selection": {
    "nodeId": "root",
    "topic": "根节点"
  },
  "schema": "..."
}
```

（上例为 **Mindmap Desktop**；扩展宿主请将 `editor` 理解为 `mindmap-vscode`。）

**用法**：对话开始时调用，获取上下文和操作说明

---

### batch_get

**功能**：读取脑图树结构

**参数**：
- `filePath`（可选，字符串）：文件路径（实时桥接中忽略，使用当前打开的面板）
- `nodeIds`（可选，字符串数组）：节点 ID 列表，仅返回这些节点及其祖先路径
- `readDepth`（可选，数字）：最大深度（默认 64），超过深度的子节点显示为占位符
- `patterns`（可选，对象数组）：正则匹配模式，返回匹配节点的子树

**返回**：
```json
{
  "id": "root",
  "topic": "根节点",
  "children": [
    {
      "id": "n_1",
      "topic": "子节点",
      "children": []
    }
  ]
}
```

**用法**：读取全部或部分脑图内容

---

### batch_design

**功能**：批量写操作（增删改查）

**参数**：
- `operations`（必填，JSON 字符串）：操作数组
- `dryRun`（可选，布尔，默认 false）：试运行，不实际修改
- `transaction`（可选，布尔，默认 true）：事务模式，任一操作失败则回滚
- `strict`（可选，布尔，默认 true）：严格模式，返回详细错误信息

**支持的操作**：
- `getTree`：获取当前树
- `getSelection`：获取选中节点
- `select`：选中指定节点（参数：`nodeId`）
- `add`：添加子节点（参数：`parentId`、`topic`、`nodeId`（可选））
- `update`：更新节点标题（参数：`nodeId`、`topic`）
- `delete`：删除节点（参数：`nodeId`，根节点不可删）
- `move`：移动节点（参数：`nodeId`、`newParentId`、`before`（可选））

**示例**：
```json
{
  "operations": "[{\"action\":\"add\",\"parentId\":\"root\",\"topic\":\"新需求\"},{\"action\":\"update\",\"nodeId\":\"root\",\"topic\":\"项目总览\"}]",
  "transaction": true,
  "strict": true
}
```

**返回**：
```json
{
  "ok": true,
  "result": {
    "results": [
      {"ok": true, "nodeId": "n_2"},
      {"ok": true}
    ],
    "tree": {...}
  }
}
```

**用法**：修改脑图结构，建议每次调用不超过 25 个操作

## 故障排查

### 问题 1：无法连接桥接

**错误信息**：
```
无法连接 Mindmap 桥接 (http://127.0.0.1:58741)：...
```

**解决方案**：
1. **Mindmap Desktop（推荐）**：确认桌面应用已启动且主窗口未关闭；桥随应用启动。勿与扩展同时占 `58741`。
2. **扩展路径（兼容）**：确认 VS Code/Cursor 已加载 Mindmap 扩展；`mindmap.mcpBridge.enable` 为 `true`；输出面板选 **Mindmap** 查看桥是否启动。
3. 确认端口 **58741** 未被其他程序占用。
4. 重启桌面应用或 VS Code/Cursor（依你所用宿主而定）。

---

### 问题 2：Token 验证失败

**错误信息**：
```
bridge error: token mismatch
```

**解决方案**：
1. **桌面**：顶区右键 **「复制 MCP 连接信息（到剪贴板）」**，用最新 `MINDMAP_BRIDGE_TOKEN` 更新 MCP 子进程 / Router 环境变量。  
2. **扩展（兼容）**：重新执行 **`Mindmap: Show MCP Bridge Info`** 复制 token。  
3. 更新客户端配置中的 **`MINDMAP_BRIDGE_TOKEN`**，与当前宿主一致。  
4. 重启 MCP 客户端（如 Claude Desktop）或重连 MCP Router 子服务。  
5. 确认配置文件格式正确（JSON 语法）。

---

### 问题 3：找不到 MCP 服务器文件

**错误信息**：
```
未找到 MCP 入口：.../mcp-server/dist/index.js
```

**解决方案**：
1. **桌面 / 仓库**：使用仓库 `mindmap_vscode/mcp-server/dist/index.js`（需先 `npm run compile` 等构建出 `dist`），或桌面安装包内 `resources/mindmap-app/mcp-server/dist/index.js`。  
2. **扩展目录**：确认扩展已完整安装（含 `mcp-server`），`args` 中版本号与 `av-ai-dev.mindmap-vscode-*` 文件夹名一致。  
3. 重新构建扩展并安装：  
   ```bash
   cd mindmap_vscode
   python build.py
   python install.py
   ```  
4. 验证文件存在：  
   ```bash
   # Windows
   dir "%USERPROFILE%\.vscode\extensions\av-ai-dev.mindmap-vscode-*\mcp-server\dist\index.js"
   
   # macOS/Linux
   ls ~/.vscode/extensions/av-ai-dev.mindmap-vscode-*/mcp-server/dist/index.js
   ```

---

### 问题 4：Node.js 未找到

**错误信息**：
```
command not found: node
```

**解决方案**：
1. 确认 Node.js 已安装：`node --version`
2. 确认 Node.js 在 PATH 中
3. 重启终端/IDE 使 PATH 生效
4. 如果使用 nvm，确保已激活 Node.js 版本

---

### 问题 5：操作失败或返回错误

**错误信息**：
```
bridge error: ...
```

**解决方案**：
1. 确认脑图文件已在 VS Code/Cursor 中打开
2. 确认操作参数正确（参考上方工具说明）
3. 检查 `operations` 是否为有效的 JSON 字符串
4. 查看 VS Code 输出面板（Mindmap）的详细日志
5. 尝试简化操作，逐步测试

---

## 安全考虑

1. **本机限制**：HTTP 桥接仅监听 `127.0.0.1`，不暴露到网络
2. **Token 认证**：所有请求需要有效的 token
3. **Token 存储**：Token 存储在配置文件中，确保文件权限正确
4. **不要在公共环境使用**：避免在共享计算机上配置 MCP 服务器

## 高级配置

### 自定义端口

如果端口 58741 被占用，可以修改：

1. 打开 VS Code 设置
2. 搜索 `mindmap.mcpBridge.port`
3. 修改为其他端口（如 58742）
4. 更新 MCP 客户端配置中的 `MINDMAP_BRIDGE_URL`
5. 重启 VS Code 和 MCP 客户端

### 自定义 Token

默认情况下 token 自动生成，如需自定义：

1. 打开 VS Code 设置
2. 搜索 `mindmap.mcpBridge.token`
3. 设置自定义 token（建议使用强随机字符串）
4. 更新 MCP 客户端配置中的 `MINDMAP_BRIDGE_TOKEN`
5. 重启 VS Code 和 MCP 客户端
